## Background: how the voice EPROM phrase table works

The voice EPROM contains a self-contained phrase dispatch system:

- `0x0000–0x009E` — 8086 dispatch routine (called via `CALL FAR F000:0000`)
- `0x009F–0x0641` — Phrase index tables, 9 groups, 482 entries
- `0x0642–0xC4C1` — TMS5220 LPC bit-packed speech data

Each index entry is 3 bytes: `[phrase_id][addr_lo][addr_hi]`.
Word number = `group × 100 + phrase_id`.

The phrase index stores **only start addresses**. End addresses are derived as "start of the next phrase in address-sorted order." This is the authoritative phrase boundary.

**The phrase index is in the voice EPROM** (0x009F–0x0641), not the program EPROM. The program EPROM calls into the voice EPROM's dispatch code; the voice EPROM searches its own table.

---

## Bit direction in the LPC bitstream

The CAT-1000 ISR (INT1, program EPROM offset 0x152F) reads bytes from the voice EPROM one at a time and writes them directly to the TSP53C30 (TMS5220-compatible) at I/O port 0x180, with no transformation. The TMS5220 processes each byte **D0-first (MSB of the LPC field = bit 7 of the byte)**.

Despite this being phrased as "D0-first," the encoder stored the first LPC parameter's MSB in bit 7 of each byte. Reading bytes at positions 7→6→5→4→3→2→1→0 (MSB-first within each byte), assembling with the first bit as the MSB of the result, gives the correct parameter values.

**Correct STOP-frame scanning:** read each byte MSB-first (bit 7 first, assembling MSB-first). Called "native MSB-first" in the audit scripts.

**Synthesis path (PyTI):** `render_phrase_to_pcm(lpc_native)` bit-reverses each byte then passes to PyTI's `LPCSynthesizer`, which internally bit-reverses again. Net effect: PyTI reads the original native bytes MSB-first. This is correct.

**Calibration:**
- Word 220 "Affirmative": voice index says 180 bytes (start 0x1B32, next phrase at 0x1BE6). Native MSB-first STOP scan finds 179 bytes (1-byte off-by-1 from `bytes_consumed()` boundary). Audio confirmed correct at 800ms / 35 frames.
- Word 981 "Good Afternoon": 195 bytes, MSB stop at 195. ✓
- Word 982 "Good Evening": 149 bytes, MSB stop at 149. ✓

The 1-byte off-by-1 in the STOP scan is systematic: `bytes_consumed()` rounds the partially-consumed last byte up, so the scan result is within 1 byte of the index-derived end.

## Root cause of truncated CSV end addresses

The original CSV end addresses were generated by scanning for STOP frames using the **wrong bit direction** (LSB-first or some other direction), which found false STOP frames at earlier positions than the real ones. Example:

| Word | Correct end (index) | CSV end | Delta |
|------|--------------------|---------|----|
| 220 Affirmative | 0x1BE6 (180 B) | 0x1C11 (223 B) | +43 (has extra) |
| 218 Advise | 0x1C70 (137 B) | 0x1C11 (43 B) | **−94 (truncated)** |
| 215 Acknowledge | 0x1D1A (170 B) | 0x1C9E (46 B) | **−124 (truncated, audio = fragment)** |

Note: `0x1C11` is not a phrase start address in the voice EPROM index; it was a false STOP artifact.

Word 220 "Affirmative" happens to work correctly despite the wrong end address (0x1C11 > real end 0x1BE6), because PyTI stops at the real STOP frame at byte 179 and ignores the extra 44 bytes.

## Fixes applied this session

### Word 215 "Acknowledge"
- **Problem:** CSV end was `0x1C9E` (46 bytes). Audio sounded like a fragment (~5 frames, ~0.18 s).
- **Root cause:** CSV end was the false LSB-first STOP position. The real STOP (MSB-native) is at 169 bytes; the next phrase starts at 0x1D1A (170 bytes).
- **Fix:** `end_address` → `0x1D1A` in both `cat-1000_clips.csv` and `cat-1000_phrases.csv`.
- **Verification:** re-export produced 170 B LPC, 800 ms WAV. ✓

### Word 982 "Good Evening"
- **Problem:** CSV had `start = end = 0xC42C` (0 bytes). Export was SKIPPED entirely.
- **Root cause (two parts):**
  1. CSV end erroneously set equal to start.
  2. `LPC_DATA_END = 0xC42C` in `cat-1000_lpc_export.py` hard-capped the scan window to 0 bytes for the last phrase, so `--dump-csv` also failed.
- **Fix:** `end_address` → `0xC4C1` (149 bytes); `LPC_DATA_END` constant → `0xC4C1`.
- **Verification:** re-export produced 149 B LPC, 675 ms WAV. ✓

## Remaining 272 truncated words

All 272 truncated words have their MSB-native STOP frame **past** the CSV end address. For 263 of them the MSB-native STOP exactly matches the voice EPROM index end (within 1 byte).

Words truncated by 1–5 bytes may sound correct (audio clips only the final partial frame). Words truncated by 50–500 bytes will have clearly audible cut-off endings.

## Recommended fix strategy

For all 272 truncated words, the correct `end_address` is the voice EPROM index end (= start of next phrase in address-sorted order). This is equivalent to the MSB-native STOP position ± 1 byte for most words, and is the authoritative phrase boundary from the hardware.

**Script to generate corrected CSV:**

```python
PHRASE_GROUPS = {
    0:(0x009F,0x1C), 2:(0x00F3,0x4F), 3:(0x01E0,0x36), 4:(0x0282,0x2B),
    5:(0x0303,0x32), 6:(0x0399,0x41), 7:(0x0459,0x3D), 8:(0x0510,0x3C),
    9:(0x05C4,0x2A),
}
# Build: start_addr -> next_start_addr from voice EPROM
with open('eprom_images/cat-1000-voice_27SF512.BIN', 'rb') as f:
    voice = f.read()
entries = []
for grp,(off,cnt) in PHRASE_GROUPS.items():
    for i in range(cnt):
        base=off+i*3
        pid=voice[base]; addr=voice[base+1]|(voice[base+2]<<8)
        entries.append((grp*100+pid, addr))
sorted_starts = sorted(set(a for _,a in entries if 0x0642<=a<0x10000))
def index_end(s):
    i=sorted_starts.index(s)
    return sorted_starts[i+1] if i+1<len(sorted_starts) else 0xC4C1
```

For each row in `cat-1000_clips.csv`, if `start_addr` is in `sorted_starts` and `end_addr < index_end(start_addr)`, update `end_addr = index_end(start_addr)`.

## Key audit script

The following function correctly finds the TMS5220 STOP frame byte position in native EPROM data (MSB-first reading, unvoiced frames read only K1–K4):

```python
def find_stop_msb(native_data):
    """Returns bytes consumed up to and including the STOP frame, or None."""
    bp=0; bitp=7
    def rb(n):
        nonlocal bp, bitp
        v=0
        for _ in range(n):
            if bp>=len(native_data): return None
            v=(v<<1)|((native_data[bp]>>bitp)&1)
            bitp-=1
            if bitp<0: bitp=7; bp+=1
        return v
    def bc(): return bp+(0 if bitp==7 else 1)
    while True:
        e=rb(4)
        if e is None: return None
        if e==15: return bc()
        if e==0: continue
        rep=rb(1); pitch=rb(6)
        if rep is None or pitch is None: return None
        if not rep:
            for w in ([5,5,4,4] if pitch==0 else [5,5,4,4,4,4,4,3,3,3]):
                if rb(w) is None: return None
```
